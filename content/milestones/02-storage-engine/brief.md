# Milestone 2: Storage Engine

## Overview

Replace the naive "rewrite everything on every Put" approach from Milestone 1 with a proper storage engine. You'll implement a write-ahead log (WAL) and an append-only file format that supports efficient writes without rewriting the entire dataset.

## Learning Objectives

- Write-ahead logging and crash recovery
- Append-only file formats and compaction
- The trade-off between write amplification and read performance
